# Учебные задания

Репозиторий с материалами и решениями по нескольким предметам семестра (заочное обучение).

## Структура

- `tasks.md` — общий трекер заданий по всем предметам с чекбоксами.
- `internet-programming/` — Интернет программирование (Задание 1/2/3 + КР).
- Остальные предметы добавляются по мере работы (`databases/`, `info-security/` и т.п.).

## Таск-трекер: `tasks.md`

**Перед началом работы — читай.** В `tasks.md` сгруппированы все предметы с дедлайнами, исходными требованиями преподавателей и списками чекбоксов `### Прогресс`.

**Правила работы с файлом:**

1. **Отмечать сразу.** Закрыл подзадачу — поменяй `- [ ]` на `- [x]` в том же коммите, что и сами изменения по этой задаче.
2. **Описания не редактировать.** Текстовые блоки под заголовками предметов (между заголовком предмета и `### Прогресс`) — копии из исходного таск-листа, должны оставаться дословными.
3. **Новые пункты — в `### Прогресс`.** Если препод прислал доп.требование или появилась новая подзадача — добавляй чекбокс в нужный раздел `### Прогресс`. В описание препода руками лезть не нужно.
4. **Новый предмет** добавляется новой секцией по тому же шаблону: `## Название`, **Дедлайн**, описание, `### Прогресс` с чекбоксами.

## Конвенции по коду (Интернет программирование)

- Ванильный HTML/CSS/JS, без фреймворков и сборщиков.
- JS — простые обработчики `onclick="func()"` / `onchange="func(event)"`, `getElementById`. Функции в одном `<script>` в конце `<body>`.
- CSS — только теговые и одноклассовые селекторы. Никаких `:nth-child`, `:has`, CSS-переменных, `calc()`, `clamp()`.
- Картинки — конкретные стабильные URL (Unsplash, Pexels), не случайные генераторы (`loremflickr`, `picsum.photos` без seed).
- Каждое задание — своя папка `task X/` с `index.html` и подпапкой `css/`.

## Workflow

- Ветка разработки: `claude/review-shared-document-UNiQE`. Все коммиты идут туда.
- Прямой пуш в `main` блокируется (403). Чтобы влить — создай PR через GitHub MCP (`mcp__github__create_pull_request`) и сразу мерджи (`mcp__github__merge_pull_request`, `merge_method: merge`).
- GitHub Pages развёрнут с `main`, source `branch: main / (root)`. Корневой `index.html` — meta-refresh редирект на `internet-programming/task 1/index.html` (текущая «главная»).

## Чего избегать

- Сложных CSS-селекторов и фреймворков — задание простое, код должен защищаться без запинки.
- Случайных/динамических картинок (`loremflickr`, рандомные генераторы).
- Объёмных описательных комментариев в коде.
- Прямого пуша в `main`.
- Переписывания текстов требований преподавателей в `tasks.md`.
